# AI 知识库 - 项目愿景 v0.1

## 要做什么

- 每天采集 AI/LLM/Agent 相关技术动态，优先覆盖 GitHub Trending、Hacker News、arXiv 和高质量 RSS。
- 将原始内容保存到 `knowledge/raw/`，保留来源、标题、链接、采集时间和原始摘要。
- 使用 AI 完成价值分析、标签归类、中文摘要、适用人群判断和相关度评分。
- 将审核后的知识条目保存到 `knowledge/articles/`，为日报、飞书、Telegram 或后续检索做准备。
- 通过 Agent、Skill、Hook、MCP、成本控制和 LangGraph 工作流逐步演进，而不是一次性堆复杂功能。

## 不做什么

- 不做通用爬虫平台，只服务 AI 技术知识库。
- 不做社交媒体全网监控，先聚焦开发者和研究者常用信息源。
- 不做复杂前端后台，当前阶段以命令行、JSON 文件和测试为主。
- 不自动发布未经审核的内容。
- 不追求一次生成长文，优先保证事实可追溯、结构稳定、成本可控。

## 边界与验收

- 每条知识必须有真实可访问的 `source_url`。
- 每条知识必须符合 `AGENTS.md` 里的 JSON 字段约定。
- 采集、分析、整理三个阶段可以单独运行，也可以通过流水线串联。
- 失败时要记录原因，不能静默跳过。
- 至少提供格式校验、质量评分、成本统计、评估测试和安全检查。

## 怎么验证

- 运行 JSON 校验脚本，确认 `knowledge/articles/*.json` 字段完整、类型正确。
- 运行质量评分脚本，确认摘要长度、标签数量、评分范围和状态值合规。
- 运行端到端流水线，输入少量样例数据，输出可审核的知识条目。
- 运行 pytest，验证模型客户端、成本控制、路由、LangGraph 状态流转和安全策略。
- 对比有 `AGENTS.md` 与无 `AGENTS.md` 时 AI 输出是否更稳定、更符合项目约束。
